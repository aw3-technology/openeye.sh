Every time a typical robot perception system processes a frame, it starts from scratch. No memory of the previous frame's context. No knowledge of the environment's layout. No recollection that the last time it saw this configuration of objects, something went wrong.

This is like asking a human to navigate their kitchen blindfolded, but also with amnesia. Every time they open their eyes for a split second, they see the kitchen fresh — no spatial memory, no object permanence, no learned patterns. It's absurd for humans, and it's limiting for robots.

## The Problem with Stateless Perception

Stateless perception — processing each frame independently — works fine for simple detection tasks. "Is there a person in this frame? Yes or no." But it fails dramatically for anything that requires temporal context:

**Object tracking**: Without memory, the system can't distinguish between an object that moved and a new object that appeared. A cup that slides from position A to position B looks like cup_A disappeared and cup_B materialized.

**Anomaly detection**: How do you know something is unusual if you don't know what's usual? A box in the middle of a hallway might be normal (it's always there) or anomalous (it appeared 30 seconds ago). Without memory, every observation is equally novel.

**Environment learning**: A robot that works in the same warehouse every day should get better at understanding that environment over time. It should learn the layout, know where objects typically are, understand traffic patterns. Stateless perception can't learn because it can't remember.

**Safety context**: Was this hazard already acknowledged by an operator? Has this configuration been flagged before? Without memory, the safety system either alerts on every frame (alert fatigue) or misses recurring issues.

## OpenEye's Memory Architecture

OpenEye implements persistent memory using a vector store (FAISS) that maintains a spatial-temporal representation of environments. Here's what it stores:

### Spatial Memory
Every scene graph generated by the perception pipeline gets indexed into the spatial memory store. Over time, this builds a rich model of the environment:

- Object positions and their typical locations
- Spatial relationships that persist (the coffee machine is always next to the sink)
- Surface maps (where objects can be placed, where they can't)
- Zone definitions learned from observation (high-traffic areas, storage areas, work zones)

### Temporal Memory
The system tracks how environments change over time:

- Object movement patterns (the forklift makes the same route every 20 minutes)
- Activity cycles (the break room is empty from 9-11am, busy at noon)
- Gradual changes (inventory levels, workspace organization drift)
- Event history (incidents, near-misses, operator interventions)

### Learned Patterns
The most powerful part of memory is pattern recognition. By observing many scenes over time, the system identifies:

- Normal configurations vs. abnormal ones
- Typical object co-occurrences (PPE near the workstation entrance)
- Precursor patterns (this arrangement of objects often precedes a specific event)
- Environmental invariants (these objects never move, those objects always move)

## Recall in Action

The memory system is accessed through the `recall` command:

```bash
$ openeye recall kitchen

[MEMORY] Environment: kitchen
[MEMORY] Last observed: 2 minutes ago
[MEMORY] Known objects: 14 tracked, 8 in typical positions
[MEMORY] Anomalies: coffee_maker moved 40cm from usual position
[MEMORY] Pattern: This configuration last seen 3 days ago (Tuesday morning)
```

The system doesn't just remember what's there — it remembers what's different, what's missing, and what the implications are.

## Memory-Enhanced Perception

When the perception pipeline has access to memory, every layer gets smarter:

**Detection with priors**: If memory says there's usually a fire extinguisher on this wall, the detection model gets a prior that boosts its confidence in that region. Objects that should be there but aren't get flagged.

**Scene understanding with context**: Instead of describing the scene from scratch, the LLM layer can describe what changed since the last observation. "The workspace has been cleared except for the soldering iron, which has been moved from its storage hook to the active work surface."

**Safety with history**: The safety system knows this particular object configuration has caused a near-miss before. It can escalate the alert level based on historical data, not just current geometry.

**Planning with experience**: The planning layer knows which action sequences worked in this environment before and which ones failed. It can route the robot around known obstacles and use proven grasp strategies for familiar objects.

## Building Long-Term Understanding

The real power of persistent memory isn't in remembering individual facts — it's in building understanding that accumulates over time.

A robot with memory that operates in the same environment for a month develops something that looks remarkably like familiarity. It knows the layout without needing to survey. It anticipates where objects will be. It detects subtle changes that a stateless system would miss entirely.

This is the difference between a robot that works in an environment and a robot that understands an environment. The gap is memory, and it changes everything.

```bash
# Build memory from a survey
openeye watch --memorize --duration 60

# Use memory in future perception
openeye watch --reason --recall
```

Robots shouldn't forget. Now they don't have to.
